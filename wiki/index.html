Overview

ReanimateMC is a revolutionary plugin that transforms Minecraft's conventional death system. Instead of a player dying instantly when their health reaches zero, the plugin introduces an intermediary state—KO (knockout). This innovative mechanic allows for dynamic role-play and strategic decision-making by giving players a chance to be revived by teammates or executed by adversaries.

Perfect for Hardcore Survival, role-play, or tactical PVP servers, ReanimateMC injects a new level of depth into the gameplay experience.

Key Benefits

  • Enhanced roleplay opportunities
  • Strategic team-based gameplay
  • Persistent KO state even when offline
  • Distress signal system for coordination
  • Highly configurable and multilingual

New Features

🔄 Persistent KO State

KO state now persists if a player disconnects, ensuring continuity in gameplay.

🚨 Distress Signal

KO players can press the swap-hand key to send a distress signal that places a beacon and alerts allies.

⏰ Offline Timer

Offline KO timer continues counting down and will eliminate the player if it expires before they reconnect.

🐛 Crawling System

Players in KO state can toggle between prone and crawling modes for tactical movement.

Commands

/reanimatemc reload

Permission: reanimatemc.admin

Description: Reloads the plugin's configuration and language files.

/reanimatemc revive <player>

Permission: reanimatemc.revive

Description: Forcefully revives a player who is in the KO state.

/reanimatemc knockout <player>

Permission: reanimatemc.admin

Description: Forces a player to enter the KO (knockout) state.

/reanimatemc status <player>

Permission: No special permission required

Description: Displays the current state of the player (KO, revived, or normal).

/reanimatemc crawl

Permission: reanimatemc.crawl

Description: Toggles the KO player's state between being fully immobilized (prone) and crawling slowly.

/reanimatemc removeGlowingEffect <player>

Permission: Operator

Description: Force removes the glowing effect from a player.

Permissions

reanimatemc.admin

Grants access to all administrative commands, including reloading the plugin and managing player states.

reanimatemc.revive

Allows players to forcefully revive others who are in the KO state.

reanimatemc.knockout

Permits players to force a player into the KO state.

reanimatemc.status

Enables players to check the current state of any player (KO, revived, or normal).

reanimatemc.crawl

Allows players in the KO state to toggle between being fully immobilized and crawling slowly.

reanimatemc.removeGlowingEffect

Grants the ability to forcefully remove the glowing effect from a player, typically used by operators.

Installation

Requirements

  • Minecraft Server 1.20.1 or higher
  • Java 16 or higher
  • Spigot, Paper, Bukkit, or compatible server software

Installation Steps

  1. Download the latest ReanimateMC.jar from the releases page
  2. Place the jar file in your server's plugins folder
  3. Start or restart your server
  4. Configure the plugin using /reanimatemc commands
  5. Enjoy enhanced gameplay!
Note: Make sure to backup your server before installing any new plugins.

Configuration

ReanimateMC offers extensive configuration options to customize the plugin behavior to your server's needs.

Main Configuration

The main configuration file is located at plugins/ReanimateMC/config.yml

Key Configuration Options:

  • knockout.duration_seconds - Duration of KO state in seconds
  • prone.auto_crawl - Whether players automatically enter crawl mode
  • tablist.enabled - Show KO status in tab list
  • setup_completed - Initial setup completion flag

Language Files

Customize messages and text in multiple languages. Language files are located in plugins/ReanimateMC/lang/
